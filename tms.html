Token Management System

A web-based solution to streamline patient and visitor flow in hospitals with real-time token management.

Laravel JavaScript MySQL Bootstrap WebSockets

Project Overview

The Token Management System (TMS) was developed to streamline patient flow management in hospitals and clinics by automating the process of issuing, calling, and monitoring tokens. Traditionally, hospitals relied on manual token slips or verbal announcements, which often led to confusion, long waiting times, and poor patient experience.

This system digitizes the entire workflow: patients generate tokens for their respective departments, doctors call the tokens in sequence (or on priority if needed), and the lobby display updates in real-time. The application was initially built using PHP, HTML, JavaScript, and Bootstrap without npm for lightweight deployment, and later integrated with Laravel for scalability and modular design.

Key Features

Token Generation

Generate tokens for each department (OPD, Lab, Pharmacy, etc.) with unique numbering.

Lobby Display

Real-time lobby display showing live token status with visual and audio alerts.

Doctor Dashboard

Call next, random, hold, skip, or complete tokens with emergency priority support.

Reporting & Analytics

Daily token counts, average waiting times, and department-wise load reports.

Requirement Gathering & Analysis

Through discussions with hospital staff and administrators, the following requirements were identified:

Functional Requirements

  • Token generation for each department (OPD, Lab, Pharmacy, etc.)
  • Lobby display to show live token status
  • Doctor-side dashboard to call next, call random, hold, or skip tokens
  • Emergency token support (prioritized calling)
  • Real-time updates on lobby screens
  • Department & doctor-wise configuration

Non-Functional Requirements

  • Simple and user-friendly interface for non-technical hospital staff
  • Optimized for fast performance in local networks
  • Scalable to support multiple counters and departments
  • Secure handling of patient token data

System Design & Architecture

Architecture

  • PHP & Laravel MVC structure (backend)
  • Bootstrap + JavaScript (frontend UI)
  • MySQL database for storing tokens, doctors, and department data
  • Broadcasting (Laravel Echo + Reverb) for real-time lobby updates

Database Design

  • departments – stores department info (OPD, Lab, Pharmacy)
  • doctors – doctor details mapped to departments
  • tokens – token number, status (waiting, called, on hold, completed), priority flag
  • displays – lobby screens mapped to departments
  • users – admin/staff accounts

Workflow Model

  • Patient generates a token at reception/self-service kiosk
  • Token stored in DB with status = waiting
  • Doctor panel allows actions: Call Next, Call Random, Hold, Skip, Complete
  • Lobby display auto-updates via real-time broadcasting
  • Reports track daily token statistics per doctor/department

UI/UX Prototyping

  • Reception Panel – Token generation page
  • Doctor Panel – Token call controls (Next, Random, Hold, Complete)
  • Lobby Display – Full-screen token board with department-wise token status
  • Admin Panel – Department/Doctor management and reporting

Prototypes were designed with Bootstrap 5 + custom CSS, ensuring compatibility with screens of all sizes, especially hospital lobby TVs.

Development Phases

Phase 1 – Core Setup

PHP + MySQL environment configured. Database schema and migrations created. Admin authentication implemented.

Phase 2 – Token Generation & Department Setup

Token generation with unique numbering per department. Auto-reset feature for new day. Department-wise token prefix configuration.

Phase 3 – Doctor Dashboard

Controls for managing tokens: Call Next, Call Random, Hold, Skip/Complete. Emergency override to prioritize critical patients.

Phase 4 – Lobby Display & Real-Time Updates

Designed lobby screen UI with Bootstrap. Integrated real-time broadcasting (Laravel Echo + WebSockets). Auto-refresh and visual/audio alert when new token is called.

Phase 5 – Reporting & Analytics

Daily token count per doctor. Average waiting time calculation. Department-wise load reports.

Phase 6 – Testing & Optimization

Stress-tested with multiple concurrent departments. Optimized SQL queries for faster token fetching. Secured input validation to prevent misuse.

Deployment

Deployed on hospital internal Nginx server. Configured for ubantu-based hospital networks.

Outcome & Benefits

The Token Management System drastically improved patient handling efficiency in hospitals. Key benefits included:

  • Reduced waiting confusion and overcrowding at counters
  • Real-time lobby updates improved patient experience
  • Doctors gained full control over token flow
  • Hospital admins received actionable reports on department performance
  • Mobile App for doctors to manage tokens remotely
  • QR Code-based Token Scanning for patients
  • Voice Announcement Integration for token calls
